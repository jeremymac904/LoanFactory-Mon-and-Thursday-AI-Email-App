# Notebook Output To Markdown Workflow

## Purpose

Define the shared operating path from notebook research inputs to durable markdown knowledge documents.

## Source Basis

1. Root `README.md`
2. Lane source intake plans
3. Lane output standards
4. Weekly training and sales NotebookLM prompt files

## Current Status

First pass operating workflow for this repo.

## Needs Factual Validation

1. Exact future storage targets for the portal, AI Twin, or automation systems
2. Exact Gemini and NotebookLM feature availability at the time of future runs

## Needs Compliance Review

1. Any borrower, realtor, lead, or mortgage product communication generated during this flow before external distribution

## Workflow

1. Start with the lane brief, domain map, output standards, and source intake plan
2. Load the strongest primary sources and real examples into the matching notebook
3. Run the starter prompt to establish the first source grounded synthesis
4. Run refinement prompts to force build queues, workflow steps, and reusable patterns
5. Use Gemini only to reorganize or expand the notebook findings while staying source bounded
6. Use ChatGPT to normalize structure, split mixed outputs into document candidates, and identify missing sections
7. Use Codex to create or update durable markdown files in `knowledge_system/`
8. Run a validation pass for structure, uncertainty flags, and review requirements
9. Archive the refined markdown files as the canonical lane source set

## Manual Versus Assisted Boundary

Keep manual:

1. Deciding whether a source actually belongs in a lane
2. Approving final messaging tone
3. Approving borrower facing or marketing copy
4. Deciding what should become a portal asset or automation

Good candidates for assisted work:

1. Summaries
2. Pattern extraction
3. Build queue generation
4. Template shaping
5. Crosswalking notebook output into markdown sections

## Handoff Outputs

Each run should ideally produce:

1. One or more updated markdown files
2. A short gap list
3. A list of `Needs factual validation` items
4. A list of `Needs compliance review` items
5. The single next best document or template to build
