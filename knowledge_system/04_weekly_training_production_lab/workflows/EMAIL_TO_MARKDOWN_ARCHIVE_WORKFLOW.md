## Current Status

First pass workflow for preserving weekly issues as reusable markdown knowledge assets.

## Needs Factual Validation

1. Final archive folder conventions by month, quarter, or topic
2. Final metadata fields required by future search or portal systems

## Needs Compliance Review

1. Archived issues that contain sensitive or time limited guidance should be reviewed before reuse

## Workflow

1. Start from the final approved weekly issue
2. Remove send only elements that do not belong in a knowledge archive
3. Add archive metadata
4. Link the issue to its source transcript, guide, or notes
5. Link any derived portal card, video outline, or template
6. Record factual validation and compliance review status
7. Save the issue in the future archive structure

## Recommended Archive Metadata

1. Title
2. Audience
3. Source type
4. Source references
5. Date created
6. Review owner
7. Validation status
8. Compliance review status

## Archive Entry Rules

1. Keep the final lesson intact
2. Make reuse easy by keeping headings predictable
3. Preserve uncertainty instead of editing it away
4. Mark stale or time bound items clearly
