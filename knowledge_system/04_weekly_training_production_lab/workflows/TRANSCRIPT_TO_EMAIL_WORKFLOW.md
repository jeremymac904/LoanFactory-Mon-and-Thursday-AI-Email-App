## Purpose

Define the PASS 2 workflow for turning a training transcript, meeting recap, or source guide into one Monday or Thursday training issue.

## Current Status

PASS 2 working workflow for transcript-based draft generation and editorial review.

## Input Standard

1. Start with one transcript, recap, guide, or meeting note set
2. Identify the intended send day and audience before drafting
3. Remove obvious sensitive file detail if the source contains borrower, product, or pricing specifics

## Workflow

1. Capture
   Pull the transcript or source notes into one clean working input.
2. Distill
   Use AI to summarize the session and extract candidate lessons, but do not publish the raw output.
3. Choose The One Thing
   Human review selects the single lesson that should survive into the issue.
4. Draft
   Build the issue using `The One Thing`, one middle section, `Review note`, and `The 24 Hour Task`.
5. Review
   Tighten tone, remove extra lessons, and add uncertainty flags where needed.
6. Approve
   Jeremy approves before scheduling or send.
7. Repurpose
   Route leftovers into markdown archive, portal-card ideas, prompt assets, or video-demo notes.

## AI Assisted Tasks

1. Transcript cleanup
2. Lesson candidate extraction
3. First-pass structure and section suggestions
4. Portal-card or archive-note extraction after the main issue is approved

## Human Owned Tasks

1. Selecting `The One Thing`
2. Final script and tone refinement
3. Compliance and factual review
4. Final approval and scheduling

## Failure Modes

1. Publishing the raw AI summary instead of a real issue
2. Turning one session into ten mixed tips
3. Skipping the review note when certainty is weak
4. Losing the one useful action the reader could take next

## Needs Factual Validation

## Needs Compliance Review
