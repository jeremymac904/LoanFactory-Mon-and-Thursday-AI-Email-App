# OUTPUT STANDARDS

## Every Strong Output Should Include

1. A short executive summary
2. Clear categories or sections
3. Practical recommendations
4. A useful ranking or prioritization when possible
5. Specific ideas for markdown files
6. Specific ideas for Gems or reusable workflows when relevant
7. A short gap section that says what is missing

## Writing Standards

1. Clear
2. Tactical
3. Mortgage relevant
4. No fluff
5. Easy to turn into training content
6. Easy to copy into markdown files

## Preferred Output Shapes

1. Summary
2. Use case map
3. Ranked recommendations
4. Template candidates
5. Build queue
6. Gap report

## Avoid

1. Generic theory
2. Unclear jargon
3. Overly academic writing
4. Long walls of text with no decisions
5. Mixing unrelated domains
